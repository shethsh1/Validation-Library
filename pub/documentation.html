Getting started

Brief description

  • Simple to use dynamic checking validator for inputs
  • Ability to freely choose your options on inputs that you need validated
  • Functions made to help you retrieves current errors, active errors, and remove errors that you might not want anymore, additionally you can remove all your options.
  • If you try to submit a form with errors then the fields with the error will be focused until a valid input is given

How to set up

<script src="js/Validation.js"></script>


<form id="login">
<input type="text" name="firstName" />
  ...
</form>

<script>

  const loginForm = new Validation("#login")
    loginForm.validate({
        firstName: {
            type: "text",
            numbers: false
        }
   })
</script>

The above example should give a broad overview on how to set it up. The script url where the library is based should be at the top under links. Under scripts I gave a very basic example of a first name input under the form “#login”.

The format will always be this

const form = new Validation("form id")
  form.validate({
      name assigned by you: {
          type: type of input,
          options
      }
 })

In the first case. name is firstName, type is text, and we don’t want numbers in the input numbers: false.

If a user puts a number in the first name input then it will give an error.

API Documentation

Validate Function

  • const form = new Validation("form id")
  • form.validate(options)

The options are for each type below:

Type: text

  • required: bool, if the field cant be empty
  • minLength: int, minimum length of input
  • maxLength: int, maximum length of input
  • numbers: boolean, if it can contain numbers
  • whitespace: boolean, if it can contain whitespace
  • exclude: array of inputs, inputs that should be rejected
  • isEqual: unique name for an input, tells you if its equal to given input
  • contains: array of input, if it contains any character of that array it will give an error
  • customCheck: function called checker that returns true and false and a custom error message, Example of customCheck:
const registrationForm = new Validation("#registrationForm")
registrationForm.validate({
  firstname: {
      type: "text",
      customCheck: {
      checker: function (val) {
          if (val) {
          if (val[0] === "S" || val[0] === "s") {
              return true
          }
          }
          return false

      },
      errorMessage: "No names that start with s"
      }

  })

Type: email, phoneNumber, visa, date

  • Everything in text will also work but additionally will check if the type is in correct format

Type: password

  • Everything in text will also work
  • If there is a confirmPassword field you must set the name of the input field as well
  • the confirmPassword field must then have the name of the password field under isEqual
  • E.g.
const passwordCheck = new Validation("#passwordCheck")
passwordCheck.validate({
password: {
    type: "password",
    confirmPassword: "confirmPassword"
},
confirmPassword: {
    type: "password",
    isEqual: "password"
}
})

Type: checkbox

  • checkLimit: int, maximum number of checkboxes that can be clicked
  • checkMinimum: int, minimum number of checkboxes that must be clicked

Type: dropdown

  • exclude: array of inputs, inputs that should be rejected

Type: textarea

  • required: bool, if the field cant be empty
  • minLength: int, minimum length of input
  • maxLength: int, maximum length of input
  • numbers: boolean, if it can contain numbers
  • whitespace: boolean, if it can contain whitespace
  • exclude: array of inputs, inputs that should be rejected
  • isEqual: unique name for an input, tells you if its equal to given input
  • customCheck: function called checker that returns true and false and a custom error message,
  • contains: array of input, if it contains any character of that array it will give an error

Get Active Errors

Returns an array of objects that show how many errors an input has and the unique name set by you of the input

getActiveErrors() returns and object of input names that shows all errors

Example:

{
    "firstname": [
        "Length must be at least 2",
        "Field cannot be empty"
    ],
    "lastname": [
        "Length must be at least 3",
        "Field cannot be empty"
    ],
    "email": [
        "Field cannot be empty"
    ],
    "password": [
        "Field cannot be empty"
    ],
    "cpassword": [
        "values are not equal"
    ]
}

The above example shows that the field with name firstname has 2 errors. The length is less than 2 and the field is empty (options that were set and not passed). Similarly for the other inputs

Get Errors

Retrieves all the set options for the form

Example:

{
    "lastname": {
        "type": "text",
        "minLength": 3,
        "numbers": false,
        "whitespace": false,
        "exclude": [
            "Bob"
        ],
        "required": true
    },
    "email": {
        "type": "email",
        "required": true
    },
    "password": {
        "type": "password",
        "numbers": true,
        "required": true
    },
    "cpassword": {
        "type": "password",
        "isEqual": "password",
        "required": true
    }
}

E.g. this is the first parameter you inserted in the validate function. This is stored so you can also edit it

Edit Options

.editOptions(name, newOptions)

Given the name of the input field and a set of new options it will replace the old one with the new one

For example,

const commentForm4 = new Validation("#editOptionsExample")
commentForm4.validate({
    firstname: {
        type: "text",
        minLength: 1,
        numbers: false,
    },
    email: {
        type: "email"
    },
    phonenumber: {
        type: "phoneNumber"
    },
    comment: {
        type: "textarea",
        minLength: 2,
        maxLength: 25,
    }
})

commentForm4.editOptions("firstname", { type: "text", minLength: 1000 })

In the above example, minLength for firstname is now 1000 and there is no more numbers check. Additionally you can use this to remove specific options and add extra fields.

Remove Options

.removeOptions()

Remove all the set options.

For example,

const commentForm4 = new Validation("#editOptionsExample")
commentForm4.validate({
    firstname: {
        type: "text",
        minLength: 1,
        numbers: false,
    },
    email: {
        type: "email"
    },
    phonenumber: {
        type: "phoneNumber"
    },
    comment: {
        type: "textarea",
        minLength: 2,
        maxLength: 25,
    }
})

commentForm4.removeOptions()

The above example deleted all the validation options
